## Recommended action

### Immediate (this session — done)

- **No automatic remediation applied.** `npm audit fix` would touch the lockfile non-trivially; `npm audit fix --force` would downgrade `next-auth` from 4.24.11 to 3.29.10 (BREAKING) and is explicitly out of scope without owner sign-off.
- **Continuation tasks filed:** Q-057-NEW (Next.js upgrade plan) and Q-058-NEW (snapshot tests as preconditions for both the Next.js upgrade and S2 decomposition).

### Q-057-NEW — Next.js upgrade plan (DEFER UNTIL OWNER APPROVES)

Three viable targets, each with distinct risk surface:

| Target | Effort | Risk | Notes |
|---|---|---|---|
| `next@14.2.x` latest patch | minimal | minimal | Does **not** close most criticals — the 23 advisories span 14.x and require 14.2.40+ or 15+ for full coverage. Verify with `npm view next@14 versions` against each advisory's "fixed in" field. |
| `next@15.x` (current stable) | moderate | moderate | App Router cache semantics changed; `useSearchParams()` and Server Action behaviour shifts; React 19 required (we are on React 18). |
| `next@16.x` (what `npm audit fix --force` proposes) | high | high | Two semver-major hops; React 19 strict-mode changes; partial-prerendering default; `next.config.js` migration. **Probably out of scope without dedicated sprint.** |

**Recommended path:** Audit each of the 23 advisory `fixed in` versions against `npm view next versions --json` to find the **minimal** 14.x version that closes the criticals. If none does (likely), document the constraint and plan the 15.x or 16.x migration as its own multi-day task.

**Coordination required:**
- Q-040-NEW CSP work (nonce middleware) — re-verify against new Next.js middleware execution model.
- Q-055-NEW CSRF cookie issuance — re-verify cookie ordering and middleware short-circuits.
- Q-029 `next/image` `remotePatterns` allowlist — verify still honoured in target version.
- Any `Server Actions` usage — none observed currently but verify before upgrade.

### Q-058-NEW — Snapshot tests (preconditions for S2 + Q-057-NEW)

Capture component snapshot tests for `QuantLabPanel.tsx` (1684 LOC) and `app/backtest/page.tsx` (887 LOC) **before** S2 decomposition begins **and** before the Next.js upgrade lands. These snapshots are the regression guardrail for both refactors.
