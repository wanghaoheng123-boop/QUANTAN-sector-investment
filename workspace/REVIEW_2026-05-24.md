## 🟡 Math/Statistics concerns (3 new modules — disclose to users)

### Concern 6: Q-044 `factorAttribution.ts` is NOT Fama-French / Carhart attribution

- **File:** `lib/portfolio/factorAttribution.ts:20-39`
- **Issue:** Computes **univariate** beta per factor independently (loadings.MKT, loadings.SMB, ...). The Fama-French / Carhart 5-factor model requires a **multivariate** OLS regression because the factors are correlated; univariate betas are biased.
- The `rSquared` field is fabricated: `Math.min(0.95, Math.abs(mktBeta) * 0.5)` — this is not a coefficient of determination at any sense.
- `alpha` is single-factor (only MKT), ignoring the other four factors' contributions.
- **Cursor's comment** "full multivariate OLS deferred to Phase 16" acknowledges the limitation, but the function name and types still imply real attribution.
- **Recommendation:** Either (a) rename to `naiveFactorBetas` and mark `rSquared` `null`, or (b) implement real multivariate OLS via QR decomposition. Until either lands, **do not surface factor attribution to institutional users** as "Fama-French attribution".

### Concern 7: Q-046 `portfolio/greeks.ts` per-position weighting unclear

- **File:** `lib/portfolio/greeks.ts:24-34`
- The `delta`/`gamma`/`vega`/`theta`/`rho` fields are summed across positions. There's no explicit `× shares × 100` weighting in the aggregator.
- If the caller passes per-share Greeks (delta = 0.45 per contract), the portfolio delta would be the sum of per-contract deltas — useful, but unclear from the type signature.
- If the caller passes raw `delta`, gamma etc. unweighted, the output is meaningless for a real portfolio.
- **Recommendation:** Add JSDoc that documents the contract: "delta/gamma/.../rho fields are pre-weighted by contract multiplier × shares; this function sums them verbatim". OR change the function to accept raw Greeks + shares + multiplier and do the weighting internally.

### Concern 8: Q-043 `scenarios/engine.ts` Greeks aggregation is muddled

- **File:** `lib/scenarios/engine.ts:41-63`
- The returned `.greeks` field mixes pre-shock and post-shock quantities:
  - `g.delta += (p.delta ?? p.shares) * (1 + shock.spotPct)` — scales delta by `(1 + spotPct)`, which is neither the post-shock delta (that'd come from gamma) nor a P&L contribution.
  - `g.gamma += (p.gamma ?? 0) * shock.spotPct ** 2` — this computes a P&L number (½ × Γ × dS² without the ½), but stores it in the `gamma` field of the output.
- The `portfolioPnl` field is delta × dS only — ignores gamma, vega, theta, rho contributions entirely. So the actual stress P&L is wrong for any options portfolio.
- **Recommendation:** Either (a) document this as "linear delta-only stress test, Greeks shown for reference only", or (b) implement the proper Taylor-expansion P&L:
  ```
  pnl_i = delta_i × dS + 0.5 × gamma_i × dS² + vega_i × dVol + theta_i × dT + rho_i × dR
  ```
  and report Greeks as `Σ position Greeks` (unmodified).
